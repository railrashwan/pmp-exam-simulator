## Design Context

### Users
Arabic-speaking professionals preparing for the PMP certification exam. They study seriously, often in focused sessions, and will take the real exam on Pearson VUE. The interface must feel familiar to that platform — not a study app, not a game, but a real exam environment that builds confidence through repeated exposure to the same look and feel.

### Brand Personality
**Professional. Clear. Motivating.**

The emotional goal is confidence: users should feel capable and prepared every time they open the app. The UI reinforces that by being calm, distraction-free, and authoritative — never playful, gamified, or visually noisy. When the exam ends and they pass, the interface should feel like it prepared them for the real thing.

### Aesthetic Direction
**Reference: Pearson VUE exam interface — replicate the feel, not the brand.**

The real PMP exam is delivered on Pearson VUE. This app should look and feel like a direct simulation of that environment so that on exam day, users feel at home.

- White primary backgrounds, light gray (#F5F7FA) for secondary panels
- A single professional blue runs everything — Pearson Chambray (#364395) for structure, #047b9c for interactive elements
- No per-feature color coding; exam cards use the same primary, differentiated only by label/content
- Clean borders (#D0D5DD), no heavy shadows, no gradients on functional elements
- Typography: Open Sans (Google Fonts) for English — the same font Pearson uses — Noto Sans Arabic for Arabic
- Font sizes: 17px question text, 15px options/body, 13px labels/meta — professional and readable at full-screen, not oversized
  - The original 24px base was chosen for readability but reads more like an accessibility setting than a professional exam tool; 17px question / 15px options is the right professional balance while remaining comfortable for Arabic text
- No colored card headers per exam type — use neutral structure, differentiate with subtle labels
- Light mode AND dark mode supported via CSS variables

**Anti-references:** Duolingo-style gamification, colorful dashboards, anything with emojis, gradients on backgrounds, per-feature accent colors, neon or saturated palettes.

### Design Principles

1. **Simulate the real exam, not a study app.** Every screen — the question layout, the timer, the navigation, the results — should feel like Pearson VUE. This builds muscle memory and reduces exam-day anxiety.

2. **One primary color, used with discipline.** No per-card color personalities. The Chambray blue (#364395) anchors structure; the interactive teal (#047b9c) signals clickability. That's the full palette for functional UI.

3. **Typography serves the reader, not the designer.** Question text at 17px, options at 15px. Open Sans for English, Noto Sans Arabic for Arabic. RTL layout must be pixel-perfect — Arabic users are the primary audience.

4. **Calm and distraction-free during the exam.** The exam screen should have zero decorative elements competing with the question. White space is not empty space — it is focus.

5. **Motivating without being cheerful.** Results pages can use affirming language and clear pass/fail indicators, but no confetti, no trophies, no gamification. A clean green "Passed" is more motivating to a serious professional than an animation.
